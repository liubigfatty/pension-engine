# 一、用户输入参数（小程序端）

| 序号 | 参数名称 | 必填 | 说明 |
|------|---------|------|------|
| 1 | **省份** | 是 | 自动获取该省计发基数 |
| 2 | **参保类型** | 是 | 企业职工 / 机关事业单位 / 灵活就业 |
| 3 | **性别** | 是 | 影响计发月数（男女不同退休年龄） |
| 4 | **退休年龄** | 是 | 50岁/55岁/60岁/65岁 |
| 5 | **缴费年限（年）** | 是 | 含视同缴费的总年限，支持小数 |
| 6 | **视同缴费年限（年）** | 条件必填 | 1996年6月30日前工龄，如有则必填 |
| 7 | **平均缴费指数** | 是 | 范围0.6~3.0，表示缴费档次 |
| 8 | **个人账户累计余额（元）** | 是 | 个人养老金账户存储额 |

# 七、计算引擎逻辑（Python伪代码）

```python
# === 输入参数 ===
省份 = "黑龙江"
退休年龄 = 60  # 50/55/60/65
性别 = "男"
缴费年限 = 40.0  # 总年限（含视同），支持小数
视同缴费年限 = 15.0  # 如有
平均缴费指数 = 1.0  # 范围 0.6 ~ 3.0
个人账户余额 = 100000  # 元

# === 系统参数 ===
计发基数 = 7570  # 2025年，2026年预测7705
计发月数 = 139  # 60岁退休
过渡系数 = 1.2%
御寒津贴 = 45  # 哈尔滨特有，其他高寒地区另定
艰苦边远补贴 = 0  # 非艰苦边远地区为0

# === 计算公式 ===
基础养老金 = 计发基数 × (1 + 平均缴费指数) ÷ 2 × 缴费年限 × 1%

个人账户养老金 = 个人账户余额 ÷ 计发月数

过渡性养老金 = 计发基数 × 平均缴费指数 × 视同缴费年限 × 过渡系数

基本养老金 = 基础养老金 + 个人账户养老金 + 过渡性养老金

实际发放 = 基本养老金 + 御寒津贴 + 艰苦边远补贴  # 黑龙江特有
```
