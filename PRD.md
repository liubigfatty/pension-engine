# 国家企业职工基本养老金测算平台 — 产品需求文档

> 版本：v3.0  
> 编制日期：2026-05-13  
> 编制角色：产品经理（PM）  
> 定位：全国企业职工基本养老金在线测算工具

---

## 一、产品定位

### 1.1 产品定义

**为国家企业职工提供在线基本养老金测算服务，支持全国各省份，零门槛、无需注册、无需填报个人身份信息，即开即用。**

### 1.2 核心价值

| 维度 | 价值阐述 |
|------|---------|
| **用户价值** | 零门槛、无需注册、无需填报个人身份信息、即开即用、测算结果权威准确 |
| **技术价值** | 参数化驱动架构，新增省份仅需配置文件，无需修改计算逻辑 |
| **社会价值** | 降低公众获取养老金信息的门槛，提升社保政策知晓度 |

### 1.3 适用人群

```
┌──────────────────────────────────────────────────┐
│  类别 1：临近退休人员                              │
│  年龄：45-60 岁                                  │
│  需求：测算本人基本养老金水平                      │
│  特点：对操作复杂度敏感，偏好简便操作方式          │
├──────────────────────────────────────────────────┤
│  类别 2：灵活就业人员                              │
│  年龄：30-50 岁                                  │
│  需求：测算不同缴费水平对养老金的影响              │
│  特点：需辅助工具进行缴费方案比较                  │
├──────────────────────────────────────────────────┤
│  类别 3：社保/人力资源从业人员                      │
│  年龄：25-45 岁                                  │
│  需求：为服务对象快速测算养老金                    │
│  特点：追求测算效率与准确度                        │
└──────────────────────────────────────────────────┘
```

## 二、商业模式

### 2.1 商业路径

```
第一级：免费测算服务（核心引流）
  ↓
第二级：报告导出服务（增值变现）
  ↓
第三级：专业咨询服务（深度服务变现）
```

### 2.2 免费测算服务（核心引流）

**用户操作流程：**

```
进入平台 → 选择参保地区 → 填报参保人员信息 → 提交测算
  ↓
展示结果：
  ├─ 基本养老金总额（元/月）
  ├─ 基础养老金（元/月）
  ├─ 增发基础养老金（元/月）
  ├─ 个人账户养老金（元/月）
  └─ 过渡性养老金（元/月）
  └─ 弹性提前退休测算对比
```

**免费展示内容：**
- 基本养老金总额
- 四部分明细金额
- 弹性提前退休测算对比
- 部分关键参数说明

### 2.3 报告导出服务（增值变现）

**导出报告包含：**

| 报告内容 | 说明 |
|---------|------|
| 完整参数明细 | 全部计算参数完整展示 |
| 个性化退休规划建议 | 基于测算结果的优化建议 |
| 缴费水平优化建议 | 不同缴费水平对比分析 |
| 退休地选择建议 | 不同退休地养老金差异分析 |
| 官方格式报告 | 仿社保部门标准格式 |
| 平台标识 | 报告页眉品牌标识 |

**定价策略：**

| 方案 | 定价 | 说明 |
|------|------|------|
| 单次导出 | 3 元/份 | 单次报告导出 |
| 月卡 | 9.9 元/月 | 3 份报告导出额度 |
| 年卡 | 39.9 元/年 | 12 份报告导出额度 + 社群准入资格 |

### 2.4 专业咨询服务（深度服务变现）

**服务权益：**

| 权益 | 说明 |
|------|------|
| 政策解读 | 最新养老保险政策专题解读（每周更新） |
| 个案咨询 | 针对个人参保情况提供建议（每月 1-2 次） |
| 退休方案定制 | 不同退休方案对比分析 |
| 养老保险对比 | 基本养老保险 vs 商业养老保险对比 |
| 同行交流平台 | 参保人员经验分享 |

**定价策略：**

| 方案 | 定价 | 说明 |
|------|------|------|
| 基础会员 | 99 元/年 | 政策解读 + 每月咨询 1 次 |
| 高级会员 | 199 元/年 | 政策解读 + 每月咨询 2 次 + 方案定制 |

### 2.5 用户转化路径

```
关注公众号文章 → 扫码进入小程序测算（免费）
  → 查看测算结果
  → 引导导出详细报告（3 元）
  → 导出后推荐加入专业咨询群（99 元/年）
  → 社群服务 → 续费 / 推荐他人
```

## 三、功能范围（MVP）

### 3.1 功能清单

| 功能模块 | MVP 范围 | 复杂度 | 优先级 |
|---------|---------|--------|--------|
| 基本养老金测算 | 四部分 + 弹性提前退休测算 + 参数明细 | 高 | P0 |
| 缴费水平测算 | 手工录入 + 结果展示 | 低 | P0 |
| 法定退休年龄查询 | 渐进式延迟法定退休年龄查询 | 低 | P0 |
| 参保地区选择 | 首页选择参保统筹地区 | 极低 | P0 |
| 社保卡凭证识别 | 拍照识别缴费记录 | 中 | P1 |
| 缴费凭证导入 | PDF/Excel 凭证导入 | 中 | P1 |
| 报告导出 | PDF/长图格式报告导出 | 中 | P1 |
| 测算结果分享 | 免费分享部分测算结果 | 低 | P1 |
| 测算历史记录 | 匿名 OpenID 记录 | 中 | P2 |
| 多方案对比 | 不同退休方案并排对比 | 高 | P3 |
| 社群入口 | 咨询服务引导 | 极低 | P2 |

### 3.2 MVP 范围定义

**第一阶段仅实现以下三项功能：**

| 功能 | 功能说明 |
|------|---------|
| **基本养老金测算** | 填报参保人员信息 → 选择参保地区 → 测算养老金（四部分 + 弹性提前退休测算） |
| **缴费水平测算** | 手工录入历年缴费基数 → 测算平均缴费工资指数 + 个人账户累计储存额 |
| **法定退休年龄查询** | 输入出生年月 / 性别 → 查询渐进式延迟法定退休年龄 |

**第一阶段不做：**
- ❌ 多方案对比
- ❌ 高级社群功能
- ❌ 复杂历史记录

### 3.3 参保地区支持计划

| 参保地区 | 第一阶段 | 后续阶段 |
|---------|---------|---------|
| 吉林省 | ✅ 必做 | — |
| 广东省 | ⏳ 第二阶段 | 南方参保地区代表 |
| 山东省 | ⏳ 第二阶段 | 北方参保地区代表 |
| 其他省份 | ⏳ 第三阶段 | 按需添加 |

## 五、计算逻辑设计

### 5.1 基本养老金测算 — 四部分统一框架

```
月基本养老金 = 基础养老金 + 增发基础养老金 + 个人账户养老金 + 过渡性养老金
```

| 模块 | 全国统一逻辑 | 参保地区差异参数 |
|------|-------------|-----------------|
| **基础养老金** | (退休地计发基数 + 全省计发基数 × 指数) / 2 × 累计缴费年限 × 1% | 退休地计发基数、全省计发基数 |
| **增发基础养老金** | 超过 20 年部分分段计算 | 触发条件、分段系数、计算基数选择 |
| **个人账户养老金** | 个人账户累计储存额 ÷ 计发月数 | 计发月数表（全国统一）、记账利率（各省不同） |
| **过渡性养老金** | 全省计发基数 × 视同缴费年限 × 指数 × 系数 | 过渡性系数、视同缴费起算日期 |

### 5.2 参保地区配置文件模板

```json
{
  "province": "jilin",
  "name": "吉林省",
  "account_start": "1995-07",
  "viewing_start": "1995-07",
  "base": {
    "provincial": { "1995": 369.17, ... },
    "city_specific": {
      "changchun": { "separate_until": 2030, "2020": 6927.75, "2021": 7297.75, "2022": 7604.25, "2023": 7726.67, "2024": 7852.58, "2025": 7978.25 }
    }
  },
  "modules": {
    "basic_pension": { "enabled": true, "rate_per_year": 0.01 },
    "extra_pension": {
      "enabled": true,
      "trigger": { "type": "actual_years", "threshold": 20 },
      "brackets": [
        { "from": 21, "to": 25, "rate": 0.0015 },
        { "from": 26, "to": 30, "rate": 0.0020 },
        { "from": 31, "to": null, "rate": 0.0025 }
      ]
    },
    "personal_account": {
      "enabled": true,
      "interest_rates": { "2005": 0.0226, ... }
    },
    "transitional_pension": {
      "enabled": true,
      "coefficient_over_20": 0.014,
      "coefficient_under_20": 0.012
    },
    "special_addition": { "enabled": false }
  }
}
```

### 5.3 测算流程

```
填报信息 → 解析参数 → 调用引擎 → 返回测算结果
                              ├─ 基础养老金
                              ├─ 增发基础养老金
                              ├─ 个人账户养老金
                              └─ 过渡性养老金
```

**关键设计：同一组输入数据，无论在小程序端还是网页端，测算结果必须完全一致。**

## 六、UI 设计规范

### 6.1 设计原则

```
原则一：简洁 — 每个页面只展示必要信息，不堆砌功能
原则二：层次 — 通过字号、颜色、间距建立清晰信息层级
原则三：无干扰 — 无广告、无弹窗、无多余装饰
原则四：可及性 — 大字体、高对比度、适合中老年用户使用
原则五：一致性 — 全局统一的色彩、字体、间距、圆角规范
```

### 6.2 色彩规范

```
主色调：深蓝 #1e3a5f（导航栏、主按钮）
强调色：蓝色 #2563eb（链接、选中状态）
辅助色：绿色 #10b981（上涨/正向数值）
辅助色：红色 #ef4444（下跌/负向数值）
中性色：
  ├─ 背景色 #f8f9fa
  ├─ 卡片色 #ffffff
  ├─ 分割线 #e5e7eb
  ├─ 次要文字 #6b7280
  └─ 主要文字 #111827

### 6.3 字体规范

```
平台：使用系统默认字体（-apple-system、PingFang SC）

  ├─ 标题：28px / 1.4 字重，深灰 #111827
