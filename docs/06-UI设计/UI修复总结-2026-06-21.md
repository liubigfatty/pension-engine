# UI修复总结 - 2026-06-21

## 📋 修复清单

### ✅ 1. 首页回退到宽松布局
**问题**：用户反馈"现在的太紧凑了"，希望回调到之前的宽松布局。

**修复**：
- 增大padding：`var(--space-md)` → `var(--space-lg)`
- 增大字号：品牌名从48rpx → 52rpx
- 增大间距：优势区域、按钮、弹窗等的padding和margin
- 增大阴影：让视觉层次更明显

**文件**：`miniprogram/pages/index/index.wxss`

---

### ✅ 2. Step1优化（选择参保地区）
**问题1**：省份选择器、地区选择器不是常显示。
**问题2**："下一步"按钮是灰色的，给人错觉。

**修复**：
- **地区选择器常显示**：
  - 未选择省份时，显示"请先选择省份"（placeholder样式）
  - 单基数省份（如河北省），显示"全省统一基数"（不可更改）
  - 多基数省份（如广东省），显示选择器（可选择"深圳市"或"全省其他地区"）
- **"下一步"按钮绿色填充**：
  - 使用 `.btn-next` class（已在step1.wxss中定义为绿色填充）
  - 一直可点击，但未选择省份时会提示用户

**文件**：`miniprogram/pages/step1/step1.wxml`

---

### ✅ 3. Step2优化（填写个人信息）
**问题1**：出生年月和参加工作时间选择器挤在左侧，右侧有大空。
**问题2**：缺少"退休方式选择"（法定年龄退休 / 弹性提前退休）。

**修复**：
- **日期选择器布局优化**：
  - 增加 `.date-row { width: 100%; }`，让选择器占满整行
  - 两个选择器（年、月）各占50%宽度
- **增加"退休方式选择"**：
  - 在"参加工作时间"之后，增加新的表单组
  - 两个选项："法定年龄退休"（默认选中）、"弹性提前退休"
  - 数据字段：`retirementType`（保存在globalData中，结果页面可以显示）

**文件**：
- `miniprogram/pages/step2/step2.wxml`
- `miniprogram/pages/step2/step2.wxss`
- `miniprogram/pages/step2/step2.js`

### ✅ 4. Step3优化（缴费信息）
**问题1**："参保地区/个人信息"摘要框没有用，应该删掉。
**问题2**：点击"手动输入"时，下面出的输入框数字被遮挡了一半。
**问题3**：个人账户余额的显示框数字也被遮挡了一半。
**问题4**："计算养老金"按钮是四方的，应该改成和上一步的样式一样（统一）。

**修复**：
- **删除摘要框**：删除"顶部信息摘要"部分（`.summary-card`）
- **修复输入框数字被遮挡**：
  - 增大padding：`.custom-input` 和 `.balance-input` 的padding从 `var(--space-md)` → `var(--space-lg)`
  - 让数字完全显示，不被遮挡
- **统一"计算养老金"按钮样式**：
  - 按钮class从 `btn-primary` → `btn-calculate`
  - 使用全圆角（`border-radius: var(--radius-round)`）
  - 表单有效时：绿色填充；表单无效时：灰色（不可点击）

**文件**：
- `miniprogram/pages/step3/step3.wxml`
- `miniprogram/pages/step3/step3.wxss`
