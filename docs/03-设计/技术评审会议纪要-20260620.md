### 🟡 P1 问题（建议修改）

#### 问题4：31省配置合并后体积过大

**位置**：第二章 2.2.3 31省配置文件合并

**问题**：
- 技术方案建议：合并31省配置成一个 `province-config.json`
- **未评估体积**：31省 × (1990-2025年社平工资) = 约31 × 36 = 1116条数据
- 预估JSON文件大小：**约500KB-1MB**

**风险**：
- 云函数包体积限制：**10MB**（但建议控制在2MB以内）
- 如果配置文件就占1MB，加上引擎代码，云函数包会很大
- 影响：云函数冷启动变慢

**技术建议**：
```
方案A：拆分配置文件（按地区）
- 华东：jiangsu.json, zhejiang.json, ...
- 华北：beijing.json, tianjin.json, ...
- 云函数根据省份代码，动态加载对应配置文件

方案B：用云存储存配置文件
- 配置文件存在云存储（/config/provice/sc.json）
- 云函数首次调用时，下载并缓存到内存
- 优点：云函数包体积小
- 缺点：首次调用慢（需要下载）

方案C：用云数据库存配置
- 存在云数据库集合 `province_config`
- 云函数实时查询
- 优点：数据更新方便
- 缺点：每次计算都要查数据库（性能差）
```

**需要你决策**：选择哪个方案？

#### 问题5：自动定位功能是否需要？

**位置**：第五章 5.2.1 首页设计

**问题**：
- 技术方案写了"自动定位当前省份"
- 但**未确认**你是否真的需要这个功能

**技术分析**：
- 自动定位需要：
  1. 调用 `wx.getLocation`（需要用户授权）
  2. 根据经纬度反查省份（需要第三方地图API，如腾讯地图）
  3. 增加小程序权限申请难度（需要"地理位置"权限）
- 用户体验：
  - 好处：首次进入自动选省份，方便
  - 坏处：弹出授权框，用户可能拒绝

**技术建议**：
```
方案A：只做手动选择（推荐）
- 首页默认显示"请选择省份"
- 用户点击后，弹出省份选择器
- 优点：简单，不需要权限申请
- 缺点：用户每次都要选（可以缓存用户选择）

方案B：自动定位 + 手动选择
- 首次进入尝试自动定位
- 如果定位失败 or 用户拒绝授权，手动选择
- 优点：体验好
- 缺点：开发复杂，需要申请权限
```

**需要你决策**：选择哪个方案？

### 🟢 P2 问题（可选优化）

#### 问题6：分享图片生成性能

**位置**：第五章 5.2.3 结果页设计

**问题**：
- 技术方案建议使用 `canvas` 生成分享图片
- 但**未评估**低端Android机的性能

**风险**：
- `canvas` 绘制复杂图片（含小程序码），可能卡顿
- Android低端机（如红米Note系列）可能崩溃

**技术建议**：
```
方案A：前端canvas生成（原方案）
- 优点：免费，不需要云函数
- 缺点：低端机可能卡顿

方案B：云函数生成分享图
- 前端调用云函数，传入计算结果
- 云函数生成图片（用 `node-canvas` 或 `puppeteer`）
- 返回图片临时链接
- 优点：性能稳定
- 缺点：消耗云函数资源

方案C：固定分享图模板
- 不做个性化分享图，只用固定模板
- 用户分享时，自动生成小程序卡片（微信自动截图）
- 优点：最简单
- 缺点：分享图不个性化
```

**建议**：v1.0先用方案C（最简单），v1.5再优化分享图

## 五、技术负责人建议

### 关于P0问题的建议

**问题1（数据库）**：建议选**方案B**（轻量级订单记录）
- 理由：用户可能需要"恢复购买"功能（换手机后找回报告）
- 实现：用云存储保存订单（不用数据库），简单轻量

**问题2（支付集成）**：建议**先上线免费版**
- 理由：首次提审，先别碰支付（容易拒）
- 等小程序跑稳了，v1.5再加支付

**问题3（PDF生成）**：建议**先用简单方案**
- 理由：v1.0重点是"计算准确"，报告可以简单点
- 方案：报告不用PDF，直接用HTML页面展示（前端生成）
- 优点：开发快，性能好
- 缺点：不能下载（只能在线查看、截图）

### 关于开发优先级的建议

**建议调整开发顺序**：
```
原方案：
1. 云函数开发（calculate）
2. 前端页面开发
3. 支付功能开发
4. 报告生成功能开发

建议方案：
1. 云函数开发（calculate）← 核心，先做
2. 前端页面开发（首页、输入页、结果页）← 免费功能
3. 分享功能开发 ← 免费功能
4. 提审（免费版）
5. 审核通过后，再开发支付 + 报告功能（v1.5）
```

**理由**：
- 首次提审，功能越简单，审核越快
- 先上线免费版，积累用户口碑
- 付费功能可以慢慢打磨
