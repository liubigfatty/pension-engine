### 1.3 技术架构决策记录（ADR）

#### ADR-001：采用云函数模式

**状态**：已确定

**背景**：
- 用户已实践验证：本地加载计算引擎+31省配置，量太大，加载失败
- 需要支持31省、1990-2025年社平工资数据

**决策**：
采用**云函数模式**，理由：
1. 计算逻辑不暴露（保护引擎代码）
2. 数据更新方便（改云端即可，不需发版）
3. 支持复杂计算（云函数资源比小程序端充足）

**替代方案**：
- 小程序端计算：已验证不可行（加载失败）

### 2.2 计算云函数（calculate）详细设计

#### 2.2.1 函数入口

```javascript
// cloud/functions/calculate/index.js
const cloud = require('wx-server-sdk')
const pensionEngine = require('./pension-engine.js')  // 计算引擎
const provinceConfig = require('./province-config.json') // 31省配置

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  const { 
    province,          // 省份代码（如 "sc" = 四川）
    参保时间,          // "1998-07"
    退休时间,          // "2026-12"
    缴费基数,          // 数组（历年基数）或 平均基数
    个人账户余额,      // 元
    视同缴费年限,      // 年（如有）
    过渡系数           // 省份默认值
  } = event
  
  try {
    // 1. 加载省份配置
    const config = provinceConfig[province]
    if (!config) {
      return { success: false, error: '省份配置不存在' }
    }
    
    // 2. 调用计算引擎
    const result = pensionEngine.calculate({
      省份: province,
      参保时间,
      退休时间,
      缴费基数,
      个人账户余额,
      视同缴费年限,
      过渡系数: 过渡系数 || config.过渡系数,
      省份配置: config
    })
    
    // 3. 返回结果
    return {
      success: true,
      data: {
        基础养老金: result.基础养老金,
        个人账户养老金: result.个人账户养老金,
        过渡性养老金: result.过渡性养老金,
        增发养老金: result.增发养老金 || 0,
        总计: result.总计,
        政策依据: config.政策文件,
        计算说明: result.计算说明
      }
    }
  } catch (error) {
    console.error('计算失败', error)
    return {
      success: false,
      error: error.message
    }
  }
}
```

#### 2.2.2 计算引擎引入方式

**方案**：直接复制 `pension-engine.js`（1804行）到云函数目录

**理由**：
1. 引擎代码已稳定（31省验证通过）
2. 不需要npm发布流程
3. 云函数包体积增加约50KB（可接受）

**文件结构**：


#### 2.2.3 31省配置文件存储方案（云存储 + 动态加载）

**用户决策**："选中哪个省就出哪个省的配置，这样加载快，体验好。"

**方案**：用云存储存配置，前端动态加载

**云存储文件结构**：
```
/province-config/sc.json   # 四川配置
/province-config/cq.json   # 重庆配置
/province-config/bj.json   # 北京配置
...
```

**配置文件格式**（单省）：
```json
{
  "省份名称": "四川",
  "社平工资": { "1990": 2856, "1991": 3108, ..., "2025": 8462 },
  "计发基数": { "2024": 8321, "2025": 8462 },
  "过渡系数": 0.014,
  "增发规则": { "独生子女": "0.1%/年", "高级职称": "0.1%/年" },
  "政策文件": ["川劳社发〔2006〕18号"]
}
```

**前端加载逻辑**：
```javascript
// utils/province-loader.js
const loadProvinceConfig = async (provinceCode) => {
  // 1. 检查本地缓存
  const cached = wx.getStorageSync(`province_${provinceCode}`)
  if (cached) {
    return cached
  }
  
  // 2. 从云存储下载
  const res = await wx.cloud.downloadFile({
    fileID: `cloud://your-env/province-config/${provinceCode}.json`
  })
  
  // 3. 读取文件内容
  const config = JSON.parse(wx.getFileSystemManager().readFileSync(res.tempFilePath, 'utf-8'))
  
  // 4. 缓存到本地
  wx.setStorageSync(`province_${provinceCode}`, config)
  
  return config
}

module.exports = { loadProvinceConfig }
```

**优点**：
- 模块化：只加载选中的省份配置
- 加载快：单个配置文件约30KB，下载时间<500ms
- 更新方便：只更新云存储文件，不需要发版

**缺点**：
- 首次加载需要下载（但可以缓存到本地）

**决策**：采用此方案 ✅

### 2.3 获取省份配置云函数（getProvinceConfig）设计

#### 2.3.1 功能

前端首次进入小程序，调用此函数获取：
1. 省份列表（用于选择器）
2. 各省份的社平工资年份范围（用于输入校验）
3. 增发规则说明（用于结果页展示）

#### 2.3.2 函数代码

```javascript
// cloud/functions/getProvinceConfig/index.js
const cloud = require('wx-server-sdk')
const provinceConfig = require('./province-config.json')

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  const { province } = event
  
  try {
    if (province) {
      // 返回单个省份配置
      const config = provinceConfig[province]
      if (!config) {
        return { success: false, error: '省份不存在' }
      }
      return {
        success: true,
        data: config
      }
    } else {
      // 返回所有省份列表（不含详细数据，减少传输量）
      const provinceList = Object.keys(provinceConfig).map(key => ({
        代码: key,
        名称: provinceConfig[key].省份名称,
        年份范围: {
          开始: Math.min(...Object.keys(provinceConfig[key].社平工资)),
          结束: Math.max(...Object.keys(provinceConfig[key].社平工资))
        }
      }))
      return {
        success: true,
        data: provinceList
      }
    }
  } catch (error) {
    return {
      success: false,
      error: error.message
    }
  }
}
```

### 2.4 支付云函数（createPayment）设计 [待用户确认细节]

#### 2.4.1 支付流程

```
前端 → 用户点击"查看详细报告"
     ↓
  弹出付费提示（9.9元）
     ↓
  用户点击"立即支付"
     ↓
  前端调用云函数 createPayment
     ↓

     ↓
  返回支付参数（timeStamp、nonceStr、package等）
     ↓
  前端调用 wx.requestPayment 发起支付
     ↓
  支付成功 → 前端调用 generateReport 生成报告
```

#### 2.4.2 关键问题 [待用户确认]

1. **微信支付商户号是否已申请？**
   - 如未申请，需要：
     - 登录微信支付商户平台（https://pay.weixin.qq.com）
     - 提交个体工商户营业执照
     - 审核周期：1-3个工作日

2. **虚拟支付资质是否已申请？**
   - 微信小程序后台 → 功能 → 虚拟支付 → 申请开通
   - 需要提交：
     - 小程序类目（工具-计算器）
     - 虚拟商品说明（详细报告PDF）
   - 审核周期：3-5个工作日

3. **支付回调如何处理？**
   - 微信支付成功后，会异步回调（需要配置回调URL）
   - **方案A**：云函数支持HTTP回调（需要确认是否支持）
   - **方案B**：用云托管（CloudBase Run）部署Express服务器
   - **建议**：v1.0先用方案A（如不支持，再改方案B）

### 2.5 报告生成云函数（generateReport）设计

#### 2.5.1 报告内容

**详细报告包含**：
1. 历年缴费基数对比表
2. 各项养老金计算公式拆解
3. 政策依据文件清单
4. 敏感性分析（延迟退休1年/3年对比）

#### 2.5.2 生成流程

```
前端 → 调用云函数 generateReport（传入计算结果）
     ↓
  云函数 → 生成PDF（使用 pdfkit）
     ↓
  上传PDF到云存储（路径：/reports/{openid}/{timestamp}.pdf）
     ↓
  返回下载链接（临时链接，有效期30分钟）
     ↓
  前端 → 展示"下载报告"按钮，点击后保存或分享
```

#### 2.5.3 PDF生成代码示例

```javascript
// cloud/functions/generateReport/index.js
const cloud = require('wx-server-sdk')
const PDFDocument = require('pdfkit')
const fs = require('fs')
const path = require('path')

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  const { 计算结果, openid } = event
  
  try {
    // 1. 创建PDF文档
    const doc = new PDFDocument()
    const fileName = `report_${openid}_${Date.now()}.pdf`
    const filePath = path.join('/tmp', fileName)
    const stream = fs.createWriteStream(filePath)
    doc.pipe(stream)
    
    // 2. 写入内容
    doc.fontSize(20).text('养老金计算详细报告', { align: 'center' })
    doc.moveDown()
    doc.fontSize(12).text(`生成时间：${new Date().toLocaleString()}`)
    doc.moveDown()
    
    // 计算结果
    doc.fontSize(16).text('一、计算结果')
    doc.fontSize(12).text(`基础养老金：${计算结果.基础养老金} 元/月`)
    doc.text(`个人账户养老金：${计算结果.个人账户养老金} 元/月`)
    doc.text(`过渡性养老金：${计算结果.过渡性养老金} 元/月`)
    doc.text(`增发养老金：${计算结果.增发养老金} 元/月`)
    doc.text(`总计：${计算结果.总计} 元/月`)
    doc.moveDown()
    
    // 政策依据
    doc.fontSize(16).text('二、政策依据')
    计算结果.政策依据.forEach(file => {
      doc.fontSize(12).text(`- ${file}`)
    })
    doc.moveDown()
    
    // 计算说明
    doc.fontSize(16).text('三、计算说明')
    doc.fontSize(12).text(计算结果.计算说明)
    
    // 3. 完成PDF生成
    doc.end()
    await new Promise(resolve => stream.on('finish', resolve))
    
    // 4. 上传到云存储
    const cloudPath = `reports/${openid}/${fileName}`
    await cloud.uploadFile({
      cloudPath,
      fileContent: fs.readFileSync(filePath)
    })
    
    // 5. 获取临时下载链接
    const downloadUrl = await cloud.getTempFileURL({
      fileList: [cloudPath]
    })
    
    // 6. 返回结果
    return {
      success: true,
      data: {
        fileId: cloudPath,
        downloadUrl: downloadUrl.fileList[0].tempFileURL
      }
    }
  } catch (error) {
    console.error('报告生成失败', error)
    return {
      success: false,
      error: error.message
    }
  }
}
```

```javascript
// utils/cloud.js - 封装云函数调用
const callCloudFunction = (name, data) => {
  return new Promise((resolve, reject) => {
    wx.cloud.callFunction({
      name,
      data,
      success: (res) => {
        if (res.result.success) {
          resolve(res.result.data)
        } else {
          reject(new Error(res.result.error))
        }
      },
      fail: (err) => {
        reject(new Error('云函数调用失败：' + err.errMsg))
      }
    })
  })
}

module.exports = { callCloudFunction }
```

#### 5.2.2 输入页（pages/input）

**功能**：
1. 输入参保时间（年份选择器）
2. 输入退休时间（年份选择器）
3. 输入缴费基数（支持"历年实际基数"或"平均基数"两种模式）
4. 输入个人账户余额（可从社保APP查询）
5. 点击"计算" → 调用云函数 `calculate`

**表单校验**：
```javascript
// pages/input/input.js
Page({
  data: {
    参保时间: '',
    退休时间: '',
    缴费基数模式: 'average',  // 'average' = 平均基数，'detail' = 历年实际
    平均缴费基数: '',
    历年缴费基数: [],
    个人账户余额: '',
    loading: false
  },
  
  async onSubmit() {
    // 1. 表单校验
    if (!this.validateForm()) {
      return
    }
    
    // 2. 显示加载动画
    this.setData({ loading: true })
    
    try {
      // 3. 调用云函数
      const result = await callCloudFunction('calculate', {
        province: this.data.province,
        参保时间: this.data.参保时间,
        退休时间: this.data.退休时间,
        缴费基数: this.data.缴费基数模式 === 'average' 
          ? this.data.平均缴费基数 
          : this.data.历年缴费基数,
        个人账户余额: this.data.个人账户余额
      })
      
      // 4. 跳转到结果页
      wx.navigateTo({
        url: '/pages/result/result?data=' + encodeURIComponent(JSON.stringify(result))
      })
    } catch (error) {
      wx.showToast({
        title: '计算失败：' + error.message,
        icon: 'none'
      })
    } finally {
      this.setData({ loading: false })
    }
  },
  
  validateForm() {
    // [待补充] 详细的表单校验逻辑
    return true
  }
})
```

## 6. 性能优化方案

### 6.1 启动优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **主包体积** | 控制在1.5MB以内（当前预计500KB） | 启动时间 < 1.5秒 |
| **首屏渲染** | 首页不使用云函数，仅展示静态内容 | 首屏渲染 < 500ms |
| **预加载** | 首页 `onLoad` 时提前调用 `getProvinceConfig` | 进入输入页时无延迟 |

### 6.2 渲染优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **setData频率** | 合并多次setData调用 | 减少JS-native桥接开销 |
| **setData数据量** | 只传递视图层需要的数据 | 每次setData < 100KB |
| **图片优化** | 分享图片使用CDN，不打包进小程序 | 减少包体积 |

### 6.3 云函数优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **冷启动优化** | 云函数预热（定时触发） | 冷启动时间 < 2秒 |
| **计算缓存** | 相同输入参数，缓存计算结果（有效期1小时） | 减少重复计算 |
| **错误重试** | 云函数调用失败时，自动重试1次 | 提高成功率 |

## 7. 安全与合规方案

### 7.1 隐私合规

**要求**（微信强制）：
1. 首次进入小程序，展示隐私协议弹窗
2. 隐私协议存储在服务器，可被访问
3. 不收集用户隐私信息（PRD已明确）

**实现方案**：
```javascript
// app.js
App({
  onLaunch() {
    // 检查是否已同意隐私协议
    const agreed = wx.getStorageSync('privacy_agreed')
    if (!agreed) {
      // 展示隐私协议弹窗
      this.showPrivacyModal()
    }
  },
  
  showPrivacyModal() {
    wx.showModal({
      title: '隐私协议',
      content: '本小程序不收集您的个人信息，仅用于养老金计算。点击"同意"继续使用。',
      confirmText: '同意',
      cancelText: '退出',
      success: (res) => {
        if (res.confirm) {
          wx.setStorageSync('privacy_agreed', true)
        } else {
          wx.exitMiniProgram()
        }
      }
    })
  }
})
```

### 7.2 内容安全

**要求**（微信强制）：
1. 用户生成内容（如反馈意见）需要审核
2. 使用微信的 `msgSecCheck` API

**实现方案**：
- v1.0不涉及用户生成内容，暂不接入
- v1.5如加入"用户反馈"功能，再接入 `msgSecCheck`

### 7.3 支付安全

**要求**：
1. 支付签名必须在服务端（云函数）完成
2. 支付回调需要验证签名（防止伪造）

**实现方案**：
- 使用微信支付官方SDK（服务端）
- 回调URL配置在微信支付商户平台

## 9. 技术风险与应对

### 9.1 风险清单

| 风险 | 影响 | 应对措施 | 负责人 |
|------|------|---------|--------|
| **云函数冷启动慢** | 用户体验差 | 云函数预热 + 加载动画优化 | 技术负责人 |
| **微信支付审核被拒** | 无法上线付费功能 | 提前申请虚拟支付资质 + 准备备选方案（广告变现） | 产品负责人 |
| **31省数据更新不及时** | 计算不准确 | 建立数据更新SOP，每年3月前完成 | 数据负责人 |
| **小程序审核被拒** | 上线延期 | 提前自查合规要求 + 准备申诉材料 | 产品负责人 |

### 9.2 应急预案

**场景1：云函数故障**
- 现象：计算请求超时
- 应对：
  1. 前端展示"服务繁忙，请稍后重试"
  2. 技术负责人排查云函数日志
  3. 如无法快速修复，启用备用云函数（提前部署）

**场景2：微信支付故障**
- 现象：用户支付失败
- 应对：
  1. 前端展示"支付通道维护中，免费提供详细报告"
  2. 技术负责人排查支付接口
  3. 如无法快速修复，临时关闭付费功能
