### 1.2 技术选型（已确定）

| 层级 | 技术选型 | 确定依据 |
|------|---------|---------|
| **前端** | 微信小程序原生开发 | 性能最优，与微信生态深度集成 |
| **计算引擎** | 云函数（Node.js） | **用户已实践验证**，本地加载量太大 |
| **配置存储** | 云存储（动态加载） | **用户决策**：模块化，选中哪个省就加载哪个省 |
| **数据库** | 云开发数据库（MongoDB） | 保存支付订单和报告下载记录（轻量级） |
| **支付** | 微信虚拟支付 | v1.5接入（v1.0免费版） |
| **报告生成** | v1.0：HTML页面；v1.5：PDF | 平衡开发速度和用户体验 |

### 1.3 技术架构决策记录（ADR）

#### ADR-001：采用云函数模式

**状态**：已确定

**背景**：
- 用户已实践验证：本地加载计算引擎+31省配置，量太大，加载失败
- 需要支持31省、1990-2025年社平工资数据

**决策**：
采用**云函数模式**，理由：
1. 计算逻辑不暴露（保护引擎代码）
2. 数据更新方便（改云端即可，不需发版）
3. 支持复杂计算（云函数资源比小程序端充足）

**替代方案**：
- 小程序端计算：已验证不可行（加载失败）

#### ADR-002：31省配置采用云存储+动态加载

**状态**：已确定（用户决策）

**背景**：
- 31省配置文件，如果合并成一个JSON，体积约500KB-1MB
- 如果全部打包进云函数，会影响冷启动速度

**决策**：
采用**云存储+动态加载**方案，理由：
1. 模块化：选中哪个省就加载哪个省的配置（约30KB）
2. 加载快：只加载需要的数据
3. 更新方便：只更新云存储文件，不需要发版

**实现方案**：
- 云存储路径：`/config/sc.json`, `/config/cq.json`, ...
- 前端加载：用户选择省份后，调用 `wx.cloud.downloadFile` 下载对应配置文件
- 缓存策略：首次下载后，缓存到本地，下次直接从缓存读取

#### ADR-003：v1.0不上线支付功能

**状态**：已确定（用户决策）

**背景**：
- PRD要求：免费计算 → 付费详细报告 → 个性化咨询
- 但微信虚拟支付资质需要申请（审核周期3-5个工作日）
- 首次提审，功能越简单，审核越快

**决策**：
v1.0先上线**免费版**（只做养老金计算+分享），v1.5再加支付功能，理由：
1. 首次提审容易通过（功能简单，不涉及支付）
2. 先积累用户口碑，再上线付费功能
3. 虚拟支付资质在v1.0开发期间同步申请，v1.5上线时正好可用

### 2.2 计算云函数（calculate）设计

#### 2.2.1 函数入口

```javascript
// cloud/functions/calculate/index.js
const cloud = require('wx-server-sdk')
const pensionEngine = require('./pension-engine.js')  // 计算引擎（从 养老金计算平台/engine/ 复制）

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV
})

exports.main = async (event, context) => {
  const { 
    province,          // 省份代码（如 "sc" = 四川）
    参保时间,          // "1998-07"
    退休时间,          // "2026-12"
    缴费基数,          // 数组（历年基数）或 平均基数
    个人账户余额,      // 元
    视同缴费年限,      // 年（如有）
    过渡系数           // 省份默认值
  } = event
  
  try {
    // 1. 从云存储加载省份配置
    const config = await loadProvinceConfig(province)
    
    // 2. 调用计算引擎
    const result = pensionEngine.calculate({
      省份: province,
      参保时间,
      退休时间,
      缴费基数,
      个人账户余额,
      视同缴费年限,
      过渡系数: 过渡系数 || config.过渡系数,
      省份配置: config
    })
    
    // 3. 返回结果
    return {
      success: true,
      data: {
        基础养老金: result.基础养老金,
        个人账户养老金: result.个人账户养老金,
        过渡性养老金: result.过渡性养老金,
        增发养老金: result.增发养老金 || 0,
        总计: result.总计,
        政策依据: config.政策文件,
        计算说明: result.计算说明
      }
    }
  } catch (error) {
    console.error('计算失败', error)
    return {
      success: false,
      error: error.message
    }
  }
}

// 从云存储加载省份配置
async function loadProvinceConfig(province) {
  // 1. 检查内存缓存
  if (global.provinceCache && global.provinceCache[province]) {
    return global.provinceCache[province]
  }
  
  // 2. 从云存储下载
  const res = await cloud.downloadFile({
    fileID: `cloud://your-env/config/${province}.json`
  })
  
  const config = JSON.parse(require('fs').readFileSync(res.fileContent, 'utf-8'))
  
  // 3. 缓存到内存
  if (!global.provinceCache) {
    global.provinceCache = {}
  }
  global.provinceCache[province] = config
    
  return config
}
```

#### 2.2.2 计算引擎引入方式

**方案**：直接复制 `pension-engine.js`（1804行）到云函数目录

**理由**：

2. 不需要npm发布流程
3. 云函数包体积增加约50KB（可接受）

### 4.2 关键页面设计

#### 4.2.1 首页（pages/index）

**功能**：
1. 手动选择省份（**无自动定位**，不收集用户信息）
2. 功能入口：
   - "开始计算" → 跳转到输入页
   - "查看示例" → 跳转到示例结果页

**交互逻辑**：
```javascript
// pages/index/index.js
const { loadProvinceConfig } = require('../../utils/province-loader')

Page({
  data: {
    province: '',  // 省份代码（如 "sc"）
    provinceName: '',  // 省份名称（如 "四川"）
    provinceList: []  // 省份列表（用于选择器）
  },
  
  onLoad() {
    // 1. 加载省份列表
    this.loadProvinceList()
    
    // 2. 读取缓存的省份（如果有）
    const cached = wx.getStorageSync('selected_province')
    if (cached) {
      this.setData({
        province: cached.code,
        provinceName: cached.name
      })
    }
  },
  
  async loadProvinceList() {
    try {
      const res = await wx.cloud.callFunction({
        name: 'getProvinceList'
      })
      if (res.result.success) {
        this.setData({
          provinceList: res.result.data
        })
      }
    } catch (error) {
      wx.showToast({
        title: '加载省份列表失败',
        icon: 'none'
      })
    }
  },
  
  onProvinceChange(e) {
    const index = e.detail.value
    const province = this.data.provinceList[index]
    
    this.setData({
      province: province.code,
      provinceName: province.name
    })
    
    // 缓存到本地
    wx.setStorageSync('selected_province', {
      code: province.code,
      name: province.name
    })
  },
  
  onShareAppMessage() {
    return {
      title: '养老金计算器 - 快速测算你的养老金',
      path: '/pages/index/index',
      imageUrl: '/images/share-cover.png'
    }
  }
})
```

```javascript
// pages/result/result.js
Page({
  data: {
    计算结果: null,
    showShareImage: false,
    shareImagePath: ''
  },
  
  async generateShareImage() {
    // 使用 canvas 生成分享图片
    const canvas = wx.createCanvasContext('shareCanvas')
    const width = 750
    const height = 1000
    
    // 1. 绘制背景
    canvas.setFillStyle('#ffffff')
    canvas.fillRect(0, 0, width, height)
    
    // 2. 绘制标题
    canvas.setFontSize(32)
    canvas.setFillStyle('#333333')
    canvas.fillText('我的养老金测算结果', 50, 80)
    
    // 3. 绘制计算结果
    canvas.setFontSize(24)
    canvas.setFillStyle('#ff6b6b')
    canvas.fillText(`总计：${this.data.计算结果.总计} 元/月`, 50, 160)
    
    // 4. 导出图片
    canvas.draw(false, () => {
      wx.canvasToTempFilePath({
        canvasId: 'shareCanvas',
        success: (res) => {
          this.setData({
            showShareImage: true,
            shareImagePath: res.tempFilePath
          })
        }
      })
    })
  }
})
```

## 5. 性能优化方案

### 5.1 启动优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **主包体积** | 控制在1.5MB以内（当前预计500KB） | 启动时间 < 1.5秒 |
| **首屏渲染** | 首页不使用云函数，仅展示静态内容 | 首屏渲染 < 500ms |
| **预加载** | 首页 `onLoad` 时提前调用 `getProvinceList` | 进入输入页时无延迟 |

### 5.2 渲染优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **setData频率** | 合并多次setData调用 | 减少JS-native桥接开销 |
| **setData数据量** | 只传递视图层需要的数据 | 每次setData < 100KB |
| **图片优化** | 分享图片使用CDN，不打包进小程序 | 减少包体积 |

### 5.3 云函数优化

| 优化项 | 方案 | 预期效果 |
|--------|------|---------|
| **冷启动优化** | 云函数预热（定时触发） | 冷启动时间 < 2秒 |
| **配置缓存** | 省份配置缓存到内存（global.provinceCache） | 减少云存储下载次数 |
| **计算缓存** | 相同输入参数，缓存计算结果（有效期1小时） | 减少重复计算 |

## 6. 安全与合规方案

### 6.1 隐私合规

**要求**（微信强制）：
1. 首次进入小程序，展示隐私协议弹窗
2. 隐私协议存储在服务器，可被访问
3. **不收集用户隐私信息**（PRD已明确）

**实现方案**：
```javascript
// app.js
App({
  onLaunch() {
    // 检查是否已同意隐私协议
    const agreed = wx.getStorageSync('privacy_agreed')
    if (!agreed) {
      // 展示隐私协议弹窗
      this.showPrivacyModal()
    }
  },
  
  showPrivacyModal() {
    wx.showModal({
      title: '隐私协议',
      content: '本小程序不收集您的个人信息，仅用于养老金计算。点击"同意"继续使用。',
      confirmText: '同意',
      cancelText: '退出',
      success: (res) => {
        if (res.confirm) {
          wx.setStorageSync('privacy_agreed', true)
        } else {
          wx.exitMiniProgram()
        }
      }
    })
  }
})
```

### 6.2 内容安全

**要求**（微信强制）：
1. 用户生成内容（如反馈意见）需要审核
2. 使用微信的 `msgSecCheck` API

**实现方案**：
- v1.0不涉及用户生成内容，暂不接入
- v1.5如加入"用户反馈"功能，再接入 `msgSecCheck`

### 6.3 支付安全

**要求**：
1. 支付签名必须在服务端（云函数）完成
2. 支付回调需要验证签名（防止伪造）

**实现方案**（v1.5）：
- 使用微信虚拟支付官方SDK（服务端）
- 回调URL配置在微信支付商户平台

## 8. 技术风险与应对

### 8.1 风险清单

| 风险 | 影响 | 应对措施 | 负责人 |
|------|------|---------|--------|
| **云函数冷启动慢** | 用户体验差 | 云函数预热 + 加载动画优化 | 技术负责人 |
| **微信支付审核被拒** | 无法上线付费功能 | 提前申请虚拟支付资质 + 准备备选方案（广告变现） | 产品负责人 |
| **31省数据更新不及时** | 计算不准确 | 建立数据更新SOP，每年3月前完成 | 数据负责人 |
| **小程序审核被拒** | 上线延期 | 提前自查合规要求 + 准备申诉材料 | 产品负责人 |

### 8.2 应急预案

**场景1：云函数故障**
- 现象：计算请求超时
- 应对：
  1. 前端展示"服务繁忙，请稍后重试"
  2. 技术负责人排查云函数日志
  3. 如无法快速修复，启用备用云函数（提前部署）

**场景2：微信支付故障**（v1.5）
- 现象：用户支付失败
- 应对：
  1. 前端展示"支付通道维护中，免费提供详细报告"
  2. 技术负责人排查支付接口
  3. 如无法快速修复，临时关闭付费功能
