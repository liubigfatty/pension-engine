### 联调问题记录

#### 问题1：前端 `personalAccount` 传 `"0"`（字符串）导致引擎解析错误
- **发现日期**：2026-06-18
- **问题现象**：云函数报错 `personalAccount is not a number`
- **根因**：`step3.js` 第109行 `personalAccount: 0` 写对了（数字），但早期版本传了字符串
- **修复**：确保 `personalAccount` 始终传 `Number` 类型（允许 `0` 或正整数）
- **状态**：✅ 已修复

#### 问题2：`birthDate` / `workStartDate` 格式不一致
- **发现日期**：2026-06-19
- **问题现象**：云函数解析 `birthYear` 时用了 `parseInt(birthDate)`（未按 `-` 分割）
- **根因**：前端传的是 `"1974-05"`，云函数第37行代码是按 `-` 分割的，但实际传参时字段名是 `birthDate`（字符串），解析正确
- **验证**：手动构造请求 `{ birthDate: "1974-05" }`，云函数正确解析出 `birthYear=1974`, `birthMonth=5`
- **状态**：✅ 无问题（代码已正确）

#### 问题3：`extras` 字段格式（加发项）
- **发现日期**：2026-06-20
- **问题现象**：吉林省案例有 `extra_pension` 字段，但前端 `extras` 传的是 `{ extra_pension: 286.23 }`（下划线命名）
- **根因**：引擎内部用 `input.extras.extra_pension` 读取，前端需传下划线命名
- **验证**：查看 `pension-engine.js` 第320行左右，确认引擎用 `input.extras?.extra_pension` 读取
- **状态**：✅ 已确认格式正确

```javascript
// step3.js 第112行（实际代码）
wx.cloud.callFunction({
  name: 'calculate',
  data: {
    province: 'jilin',
    cityType: 'cc',
    gender: 'female',

    if (res.result && res.result.success) {
      // 处理结果
      const result = res.result.data
      app.globalData.calcResult = result
      wx.navigateTo({ url: '/pages/result/result' })
    }
  },
  fail: (err) => {
    wx.showToast({ title: '计算失败', icon: 'none' })
  }
})
```
