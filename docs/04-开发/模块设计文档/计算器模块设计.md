# 计算器模块设计文档

**版本**：v1.0  
**日期**：2026-06-20  
**负责人**：前端负责人  
**模块**：养老金计算器（核心业务模块）

---

## 一、模块概述

**功能定位**：用户输入个人参保参数，调用云函数计算养老金，展示计算结果。

**用户流程**：
```
首页（隐私确认）→ Step1（选省份/城市类型）→ Step2（填个人信息）→ Step3（填缴费参数）→ 结果页
```

**技术架构**：
```
小程序端（输入+展示） → 云函数（计算引擎） → 省份配置（provinces/*.js）
```

---

## 二、页面结构设计

### 2.1 页面路由（`app.json`）

| 页面 | 路径 | 功能 |
|------|------|------|
| 首页 | `pages/index/index` | 隐私协议确认、入口 |
| Step1 | `pages/step1/step1` | 选择省份 + 城市类型（计发基数类型） |
| Step2 | `pages/step2/step2` | 填写性别、出生日期、参加工作时间 |
| Step3 | `pages/step3/step3` | 填写平均缴费指数、个人账户余额、加发项，触发计算 |
| 结果页 | `pages/result/result` | 展示计算结果 + 分享图 |

### 2.2 数据传递方式

**使用 `app.globalData.calcInput` 跨页面传递输入参数**（不使用 URL 参数，避免长度限制）。

```javascript
// app.js
globalData: {
  calcInput: {}   // Step1/2/3 依次写入
  calcResult: null  // Step3 写入，result 页读取
}
```

**数据流**：
```
Step1 → 写 calcInput.province / cityType
Step2 → 写 calcInput.gender / birthDate / workStartDate / identity / genderType
Step3 → 写 calcInput.averageIndex / personalAccount / extras
Step3 → 调用云函数 → 写 calcResult
Result ← 读 calcResult
```

## 三、云函数接口设计

### 3.1 接口：`calculate`

**触发方式**：`wx.cloud.callFunction({ name: 'calculate', data: input })`

**输入参数**（与 `cloudfunctions/calculate/index.js` 一致）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `province` | String | ✅ | 省份 code（如 `jilin`、`guangdong`） |
| `cityType` | String | ✅ | 城市类型（`prov`/`changchun`/`shenzhen` 等） |
| `gender` | String | ✅ | `male` / `female` |
| `genderType` | String | ⬜ | `male`/`fw50`/`fw55`/`fc`（女性退休年龄类型） |
| `identity` | String | ⬜ | `enterprise`/`flexible`（影响过渡性养老金） |
| `birthDate` | String | ✅ | `1990-05`（年-月） |
| `workStartDate` | String | ✅ | `2012-08`（年-月） |
| `averageIndex` | Number | ✅ | 平均缴费指数（如 `1.2`） |
| `personalAccount` | Number | ⬜ | 个人账户余额；**不填则引擎自动估算** |
| `extras` | Object | ⬜ | 加发项（如 `{ jiangsuSubsidy: 8.85 }`） |

### 3.2 输出格式

**成功**：
```json
{
  "success": true,
  "data": {
    "_raw": { "legal": {...}, "metaData": {...} },
    "totalAmount": "3882.11",
    "basePension": "2374.30",
    "personalPension": "502.48",
    "transitionPension": "719.10",
    "extraPension": "286.23",
    "workYears": "32.50",
    "accountBalance": "11525.00",
    "baseNumber": "6852.00",
    "averageIndex": "0.85",
    "months": 170,
    "retireAge": "50岁0个月",
    "birthYear": 1974,
    "birthMonth": 5
  }
}
```

**失败**：
```json
{
  "success": false,
  "message": "参数不完整"
}
```

## 四、核心模块详细设计

### 4.1 Step1 页面（`pages/step1/step1.js`）

**功能**：选择省份 + 城市类型（决定使用哪套计发基数）

**省份配置**（硬编码在 `step1.js` 顶部 `PROVINCE_CONFIG`）：
- 31 个省/自治区/直辖市
- 有独立计发基数的城市单独列出（长春、沈阳、大连、郑州、深圳）
- 配置格式：
  ```javascript
  { code: 'jilin', name: '吉林省', cityTypes: [
    { type: 'changchun', label: '长春市（独立基数）' },
    { type: 'prov', label: '全省其他地区' }
  ]}
  ```

**下一步**：写入 `app.globalData.calcInput`，跳转到 `step2`

---

### 4.2 Step2 页面（`pages/step2/step2.js`）

**功能**：填写性别、出生日期、参加工作时间，自动判断退休年龄类型

**关键逻辑**：
1. 性别选择 → 决定 `gender`（`male`/`female`）
2. 出生日期 + 参加工作日期 → 计算 `workYears`（累计缴费年限）
3. 预览退休年龄（`retirePreview` 字符串，如 `"50岁0个月"`）
4. **注**：`genderType` 判断不在 Step2，在 Step3 点击"下一步"时根据 `identity` 和 `femaleEmployeeAge` 判断

**下一步**：写入 `app.globalData.calcInput`，跳转到 `step3`

---

### 4.3 Step3 页面（`pages/step3/step3.js`）

**功能**：填写平均缴费指数、个人账户余额，触发计算

**关键逻辑**：
1. 平均缴费指数 → 必填，直接传给云函数
2. 个人账户余额 → **选填**；不填时引擎自动用复利公式估算
3. 加发项 → 根据省份动态显示（如吉林省显示"增发养老金"选项）
4. **`genderType` 判断**（`getGenderType()` 函数）：
   - `male` → `'male'`
   - `female` + `identity='flexible'` → `'fw55'`（灵活就业女性，55岁）
   - `female` + `identity='employee'` + `femaleEmployeeAge=55` → `'fc'`（女干部，55岁）
   - `female` + `identity='employee'` + `femaleEmployeeAge=50` → `'fw50'`（女工人，50岁）
5. 点击"计算" → 调用 `wx.cloud.callFunction`
6. 成功 → 结果写入 `app.globalData.calcResult`，跳转到 `result`
7. 失败 → `wx.showToast` 提示错误

**容错**：
- 网络失败 → 提示"网络错误，请重试"
- 参数错误 → 提示具体缺失字段
- 云函数超时 → 提示"计算超时，请稍后重试"

---

### 4.4 结果页（`pages/result/result.js`）

**功能**：展示计算结果，生成分享图

**数据来源**（多重容错）：
```javascript
// 优先用 step3 已格式化的数据
const totalAmount = r.totalAmount || fmt(legal.total)
const basePension = r.basePension || fmt(legal.basicPension?.amount)
// ... 每个字段都有 fallback
```

**分享图生成**（`generateShareImage()`）：
1. 使用 Canvas 2D API 绘制
2. 动态布局：过渡性养老金/增发养老金为 0 时不显示
3. 绘制完成后导出临时文件（`wx.canvasToTempFilePath`）
4. 用户可保存至相册（`wx.saveImageToPhotosAlbum`）

**分享功能**：
- `onShareAppMessage()` → 转发给微信好友
- `onShareTimeline()` → 分享到朋友圈

## 五、计算引擎设计

### 5.1 引擎入口（`cloudfunctions/calculate/pension-engine.js`）

**导出函数**：`calculate(config, input) → result`

**输入**：
- `config`：省份配置（由 `provinces/${province}.js` 的 `getEngineConfig()` 提供）
- `input`：用户输入（由云函数 `index.js` 构造）

**输出**：
```javascript
{
  legal: {
    basicPension:    { amount: Number, formula: String },
    personalAccount:  { amount: Number, formula: String },
    transitionalPension: { amount: Number, formula: String },
    extraPension:    { amount: Number, formula: String },
    total: Number
  },
  metaData: { province: String, cityType: String, ... },
  _raw: { /* 原始数据 */ }
}
```

### 5.2 省份配置（`provinces/*.js`）

**统一接口**：每个省份模块导出 `getEngineConfig()` 函数，返回引擎所需配置。

**配置内容**：
```javascript
{
  provinceKey: 'jilin',
  avg_salary_history: [  // 历年社平工资（元/年）
    { year: 1990, avg_salary: 1990 },
    ...
  ],
  base_rates: {
    prov: { '2024': 7178.5 },  // 2024年计发基数（元/月）
    cc:   { '2024': 8250.0 }   // 长春市独立基数
  },
  trans_coefficient: 1.4,  // 过渡系数（%）
  // ... 各省特殊规则
}
```

## 六、异常情况处理

| 异常场景 | 处理方式 |
|---------|---------|
| 云函数调用失败 | `wx.showToast` 提示"计算失败，请重试" |
| 网络断开 | 调用前检查 `wx.getNetworkType`，断网提示"请检查网络" |
| 参数不完整 | Step3 点击计算时校验，缺失字段 `showToast` 提示 |
| 引擎计算异常 | 云函数 `try-catch`，返回 `{ success: false, message: err.message }` |
| 分享图生成失败 | 隐藏"保存分享图"按钮，不阻塞主流程 |
| 省份配置缺失 | 云函数返回 `{ success: false, message: '未找到省份配置' }` |
