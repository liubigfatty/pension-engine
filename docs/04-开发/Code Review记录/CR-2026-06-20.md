## Issue 列表

### Issue 1：【中等】`index.js` 第35行 `genderType` 默认值逻辑可能不准确

**位置**：`cloudfunctions/calculate/index.js` 第35行

**代码**：
```javascript
genderType: genderType || (gender === "male" ? "male" : "fw50"),
```

**问题**：当 `genderType` 为空且 `gender === 'female'` 时，默认给 `'fw50'`（女工人50岁），但没有判断 `identity` 和 `femaleEmployeeAge`，可能导致灵活就业女性（应为 `fw55`）被误判为 `fw50`。

**建议**：默认逻辑应和 `step3.js` 的 `getGenderType()` 函数保持一致，或直接在云函数里调用 `getGenderType()` 等效逻辑。

**状态**：⚠️ 待确认（需验证实际是否触发此路径）

### Issue 2：【轻微】`index.js` 第42行注释拼写错误

**位置**：`cloudfunctions/calculate/index.js` 第42行

**代码**：
```javascript
personalAccInput: parseFloat(personalAccount) || 0,  // 0 或空 → 引擎自动复利估算
```

**问题**：字段名叫 `personalAccInput`，但注释写的是"引擎自动复利估算"。实际引擎内部用的是 `input.personalAccInput` 还是 `input.personalAccount`？

**验证**：查看 `pension-engine.js` 读取个人账户余额的字段名。

**状态**：⚠️ 待确认
