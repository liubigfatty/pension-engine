## 一、产品概述

### 1.1 产品定位

为普通职工提供**专业、准确、易懂**的养老金测算服务，帮助用户在退休前了解预期养老金待遇。

**产品演进路径**（三步）：
1. **第一步**：免费测算，免费分享计算结果
2. **第二步**：详细报告付费查看（9.9元）
3. **第三步**：引导进行个性化咨询（高阶服务）

### 1.2 目标用户

| 用户类型 | 特征 | 使用场景 |
|---------|------|---------|
| 临近退休人员 | 45-60岁，在职或即将退休 | 退休前测算养老金待遇 |
| 社保关注者 | 关心养老保障的在职人员 | 了解政策、规划缴费 |

### 1.3 核心价值

- **准确性**：基于31省官方政策数据和社平工资
- **专业性**：覆盖1990-2025年政策变化，支持增发规则
- **易用性**：无需注册，3步完成测算
- **可信度**：计算逻辑透明，可导出详细报告

### 1.4 产品边界

**包含范围（v1.0）**：
- 31省养老金计算（1990-2025年社平工资）
- 基础养老金 + 个人账户养老金 + 过渡性养老金
- 5省增发规则（四川、重庆、海南、云南、宁夏）
- 计算结果分享（图片/小程序卡片）
- 详细报告生成（付费）

**不包含范围（v1.0）**：
- 社保缴费记录查询（需对接社保局API，后续版本）
- 养老金待遇资格认证
- 社保转移接续办理
- 企业年金计算
- 离职测算功能（已取消）

## 二、用户场景与需求

### 2.1 用户故事

#### 场景1：快退休了，想知道能领多少养老金

**用户**：张女士，50岁，企业职工，缴费25年

**流程**：
1. 打开小程序，选择省份（默认定位或手动选择）
2. 输入基本信息：
   - 参保时间：1998年
   - 退休时间：2026年（预计）
   - 缴费基数：历年实际基数或按档次选择
   - 个人账户余额：可通过社保APP查询
3. 点击"计算"，5秒出结果
4. 查看三项养老金明细 + 总计
5. 免费分享给家人或保存截图
6. 如需详细报告，付费9.9元查看

**预期结果**：
- 清晰展示每月可领金额
- 说明计算依据（政策文件编号）
- 提供优化建议（如继续缴费可提高待遇）

---

#### 场景2：查看详细报告，了解计算细节

**用户**：王先生，58岁，高缴费基数，需要详细报告用于商业保险规划

**流程**：
1. 免费计算后，点击"查看详细报告"
2. 弹出付费提示（9.9元）
3. 微信支付完成后，生成PDF报告：
   - 历年缴费基数对比
   - 各项养老金计算公式拆解
   - 政策依据文件清单
   - 敏感性分析（延迟退休1年/3年对比）
4. 下载或分享报告
5. 引导进行个性化咨询（第三步）

**预期结果**：
- 专业报告可用于商业保险规划、财务规划
- 报告带有小程序二维码，便于分享获客
- 为个性化咨询引流

## 五、系统功能设计

### 5.1 技术架构

#### 计算模式：云函数（已实践验证）

**决策依据**：本地加载量太大，实践过多轮都加载失败，必须采用云函数模式。

| 方案 | 优点 | 缺点 | 实践结果 |
|------|------|------|----------|
| **A. 云函数模式** | 计算逻辑不暴露、易更新、支持复杂计算 | 有网络依赖、冷启动延迟 | ✅ 采用 |
| **B. 小程序端计算** | 离线可用、响应快 | 代码包体积增大、逻辑更新需发版 | ❌ 加载失败（实践验证） |

**架构设计**：
- 计算引擎部署到云函数（避免小程序端加载大量数据）
- 社平工资数据（1990-2025）放在云函数环境中
- 小程序只负责：收集参数 → 调用云函数 → 展示结果

**云函数列表**：
1. `calculate` - 养老金计算
2. `payment` - 微信支付（待确认合规性）
3. `generateReport` - 报告生成（PDF）

**合规风险提示**：
- "工具-计算器"类目是否支持虚拟商品支付，需要查证
- 如不支持，需要升级类目到"生活服务-社保"（需资质）
- 或调整方案：v1.0先做免费版，v1.5再加支付

#### 系统架构图

```
用户
 │
 ▼
微信小程序前端
 │
 ├─ 调用云函数（计算养老金）

 │
 ├─ 调用云存储（存储报告PDF）

```

**技术选型依据**：
- 本地加载量太大，实践过多轮都加载失败 ❌
- 云函数模式已验证可行 ✅
- 数据（社平工资1990-2025）放在云函数环境，避免小程序端加载

### 5.2 数据配置

#### 省份配置文件

- 位置：`provinces/*.json`（31个省）
- 内容：计发基数、过渡性系数、增发规则开关
- 更新频率：每年更新一次（社平工资公布后）

#### 社平工资数据

- 位置：`engine/data/avg_salary_history.json`
- 格式：元/年（年度总额）
- 数据来源：各省人社厅官方文件

### 5.3 接口设计（云函数）

#### 1. 计算养老金

```javascript
// 云函数入口
exports.main = async (event, context) => {
  const { province, params } = event;
  
  // 加载省份配置
  const config = require(`provinces/${province}.json`);
  
  // 调用计算引擎
  const result = calculate(config, params);
  
  return { success: true, data: result };
};
```

**输入参数**：
```json
{
  "province": "四川",
  "params": {
    "startYear": 1998,
    "retireYear": 2026,
    "paymentYears": 28,
    "baseSalaryHistory": [/* 历年缴费基数 */],
    "accountBalance": 85000,
    "hasOneChild": true,   // 四川增发
    "hasSeniorTitle": false
  }
}
```

**输出结果**：
```json
{
  "success": true,
  "data": {
    "total": 3256,
    "breakdown": {
      "basePension": 1523,
      "personalAccount": 892,
      "transitionPension": 841,
      "extraPension": 0
    },
    "policyRefs": [
      "川劳社发〔2006〕18号",
      "四川省2024年计发基数：8332元/月"
    ]
  }
}
```

#### 2. 微信支付（付费报告）

```javascript
// 云函数：创建支付订单
exports.main = async (event, context) => {
  const { reportId } = event;
  
  // 调用微信支付统一下单API
  const paymentParams = await wxPay.unifiedOrder({
    body: '养老金计算详细报告',
    out_trade_no: reportId,
    total_fee: 990,  // 9.9元 = 990分
    spbill_create_ip: context.sourceIp,
    notify_url: 'https://your-domain.com/api/pay-notify',
  });
  
  return paymentParams;  // 返回给小程序调起支付
};
```

#### 3. 生成报告

```javascript
// 云函数：生成PDF报告
exports.main = async (event, context) => {
  const { calculationResult } = event;
  
  // 使用模板引擎生成HTML
  const html = template('report.html', calculationResult);
  
  // 转PDF（使用 puppeteer 或类似库）
  const pdfBuffer = await htmlToPdf(html);
  
  // 上传到云存储
  const fileId = await cloud.uploadFile({
    cloudPath: `reports/${reportId}.pdf`,
    fileContent: pdfBuffer,
  });
  
  return { fileId };
};
```

## 七、数据需求

| 数据类型 | 来源 | 更新频率 |
|---------|------|---------|
| 社平工资（1990-2023） | 各省人社厅官方文件 | 已完成 |
