## 🚨 常见审核拒绝原因及应对

### 1. 隐私协议不符合要求

**拒绝原因示例**：
> 你的小程序未提供清晰的隐私协议，或隐私协议未说明收集的信息用途。

**应对**：
- 检查隐私协议是否包含：信息收集范围、使用目的、用户权利、联系方式
- 确保隐私协议链接可访问（HTTPS）

### 2. 类目不符

**拒绝原因示例**：
> 你的小程序实际功能与选择的类目不符。

**应对**：
- 确认类目选择正确（工具-计算器）
- 如果包含支付功能，需要升级类目（生活服务-社保）

### 3. 功能不完整

**拒绝原因示例**：
> 你的小程序存在大面积空白或"开发中"提示。

**应对**：
- 确保所有页面都有实际内容
- 不要有"开发中"或"即将上线"的提示

### 4. 诱导分享

**拒绝原因示例**：
> 你的小程序存在诱导分享行为。

**应对**：
- 不要强制用户分享才能继续使用
- 分享功能必须是用户主动触发
