# ADR-002：31省配置采用云存储动态加载

> **状态**：已决定  
> **日期**：2026-06-20  
> **决策者**：产品负责人 + 技术负责人  
> **背景**：31省配置文件较大，且需要按省份动态加载以优化体验

---

## 背景

养老金计算平台需要支持31个省的配置数据（社平工资、计发基数、增发规则等）。
现有31个省的配置文件（`provinces/*.json`），每个约10-30KB，合计约500KB-1MB。

### 核心诉求

- **模块化**：选中哪个省就加载哪个省的配置
- **加载快**：只加载需要的省份，不加载全部
- **体验好**：减少等待时间

---

## 决策

**采用云存储动态加载方案**：
- 31省配置文件存储到微信云存储
- 用户选中某省份时，动态下载该省配置文件
- 下载后缓存到本地（`wx.setStorageSync`），下次直接读取

---

## 方案对比

| 方案 | 加载速度 | 云函数包体积 | 更新方便性 | 体验 | 结论 |
|--------|----------|----------------|-------------|------|------|
| **A. 拆分配置文件**（按地区） | 快（只加载选中地区） | 小（每个文件约30KB） | 麻烦（需要更新多个文件） | 好 | ⭐⭐⭐ |
| **B. 云存储存配置**（动态加载） | 快（只加载选中省份） | 小（0KB，不占包体积） | 方便（只更新云存储文件） | **最好** | ⭐⭐⭐⭐⭐ **采纳** |
| **C. 合并成一个JSON** | 慢（需要加载所有省份配置） | 大（约500KB-1MB） | 方便（只更新一个文件） | 差 | ⭐⭐ |

---

## 决策理由

1. **完全符合诉求**：选中哪个省就加载哪个省，不浪费流量和存储
2. **包体积最优**：不占用云函数包体积（0KB）
3. **更新最方便**：更新某个省的配置，只需上传该省的JSON到云存储，无需重新发版
4. **缓存机制**：首次下载后缓存到本地，后续使用无延迟


### 前端加载逻辑

```javascript
// utils/province-config.js

// 获取省份配置（带缓存）
const getProvinceConfig = async (provinceCode) => {
  // 1. 检查本地缓存
  const cacheKey = `province_${provinceCode}`;
  const cached = wx.getStorageSync(cacheKey);
  if (cached) {
    console.log(`[配置] 从缓存加载 ${provinceCode}`);
    return cached;
  }

  // 2. 从云存储下载
  console.log(`[配置] 从云存储下载 ${provinceCode}`);
  const filePath = `province-config/${provinceCode}.json`;
  const { fileID } = await wx.cloud.downloadFile({
    fileID: `cloud://your-env-id/${filePath}`,
  });

  // 3. 读取文件内容
  const fs = wx.getFileSystemManager();
  const data = fs.readFileSync(fileID, 'utf-8');
  const config = JSON.parse(data);

  // 4. 缓存到本地
  wx.setStorageSync(cacheKey, config);

  return config;
};

// 清除缓存（用于更新配置后强制重新下载）
const clearProvinceCache = (provinceCode = null) => {
  if (provinceCode) {
    wx.removeStorageSync(`province_${provinceCode}`);
  } else {
    // 清除所有省份缓存
    const keys = wx.getStorageInfoSync().keys;
    keys.forEach(key => {
      if (key.startsWith('province_')) {
        wx.removeStorageSync(key);
      }
    });
  }
};

module.exports = { getProvinceConfig, clearProvinceCache };
```

### 云函数调用逻辑

```javascript
// cloud/functions/calculate/index.js

const cloud = require('wx-server-sdk');
const fs = require('fs');
const path = require('path');

cloud.init({
  env: cloud.DYNAMIC_CURRENT_ENV,
});

exports.main = async (event, context) => {
  const { provinceCode, ...otherParams } = event;

  try {
    // 1. 从云存储下载省份配置（云函数中也支持下载云存储文件）
    const filePath = `/tmp/province_${provinceCode}.json`;
    const downloadResult = await cloud.downloadFile({
      fileID: `cloud://your-env-id/province-config/${provinceCode}.json`,
    });
    
    // 2. 读取配置
    const configData = fs.readFileSync(downloadResult.fileContent, 'utf-8');
    const provinceConfig = JSON.parse(configData);

    // 3. 调用计算引擎
    const pensionEngine = require('./pension-engine.js');
    const result = pensionEngine.calculate({
      ...otherParams,
      provinceConfig,
    });

    return {
      success: true,
      data: result,
    };
  } catch (error) {
    return {
      success: false,
      error: error.message,
    };
  }
};
```

## 影响

### 正面影响

- ✅ **加载速度快**：只加载选中的省份，约30KB，下载时间<500ms（4G网络）
- ✅ **包体积最优**：云函数包体积不增加（配置文件在云存储）
- ✅ **更新方便**：更新某省配置，只需上传JSON到云存储，无需重新上传云函数
- ✅ **缓存优化**：首次下载后缓存，后续使用无延迟

### 负面影响

- ⚠️ **首次加载依赖网络**：无网络时无法下载新省份配置（但已缓存的省份可用）
- ⚠️ **云存储有读取次数限制**：免费额度 5万次/天（足够v1.0使用）

### 缓解措施

- 首次加载时显示loading提示
- 无网络时提示用户"请检查网络连接"
- 云存储读取次数监控（超过80%免费额度时报警）

---

## 替代方案

**方案D：配置存云数据库**
- 优点：更新方便（直接改数据库）
- 缺点：每次计算都要读数据库，性能差（延迟约100-300ms）
- **暂缓**：v1.0不用，v1.5如果配置需要频繁更新再考虑
